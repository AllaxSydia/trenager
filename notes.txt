Преимущества Docker:
    Полная изоляция

    Код в песочнице - не может повредить хост-систему

    Нет доступа к файловой системе кроме /app

    Не может делать сетевые запросы

Чистое окружение для каждого запуска
    Нет накопления состояния между запусками

    Идентичные условия для всех пользователей

    Воспроизводимые результаты

Универсальность языков
    Любой язык через официальные образы

    Не нужно устанавливать компиляторы на хост-систему

    Легко добавить новый язык

Гибкая конфигурация
    Оптимальные настройки для каждого языка

    Компиляция + выполнение в одном контейнере

Недостатки Docker:
    Производительность

    Медленный старт - 2-10 секунд на запуск

    Накладные расходы на создание/удаление контейнеров

Сложность настройки
    Требует Docker Engine на сервере

    Проблемы с правами и доступом

    Не работает на некоторых хостингах (Render.com, Heroku)

Потребление ресурсов
    Образы занимают место (Python: ~50MB, Node: ~100MB)

    Параллельные выполнения требуют много памяти

Local: Максимальная доступность
Преимущества Local:
1. ⚡ Мгновенный запуск

    Нет накладных расходов на контейнеризацию

    Идеально для быстрого тестирования

2 Простота развертывания

    Работает везде где есть базовые компиляторы

    Идеально для Render.com и подобных хостингов

3 Легкая отладка

    Прямое выполнение - проще понять ошибки

    Доступ к системным логам

4. 📈 Масштабируемость
    Минимальное потребление памяти на процесс

    Можно запускать тысячи параллельных выполнений

Недостатки Local:
1. 🚫 Ограниченная безопасность

    Может читать/писать файлы в временной директории

    Может запускать системные команды

    Риск бесконечных циклов и потребления ресурсов

2. 🏗️ Сложность поддержки языков

    Размер образа увеличивается

    Конфликты версий компиляторов

3. 🎯 Ограниченные языки
    Только предустановленные компиляторы

    Сложно добавить экзотические языки (Rust, Haskell)

    Вопрос: "Почему так быстро меняются языки?"

Простой ответ:
"У нас все языки уже установлены и готовы к работе.
Это как иметь 10 разных станков в одной мастерской -
просто переходи от одного к другому."

Техническая причина:
Все компиляторы/интерпретаторы предустановлены
Не нужно ничего скачивать или устанавливать при переключении
Быстрое создание изолированных сред выполнения

"Как это вообще работает?"
Вопрос: "Как браузер выполняет Python/Java/C++ код?"

Простой ответ:
"Браузер не выполняет код напрямую. Он отправляет его на наш сервер,
где специальная система запускает код в безопасных 'песочницах' и возвращает результат обратно."

Возможные вопросы на защите:
Q: "Почему вы используете обычный textarea, а не продвинутый редактор?"
A: "Для простоты и скорости. Можно легко заменить на Monaco Editor (как в VS Code)
когда проект вырастет."

Q: "Как вы обрабатываете синтаксические ошибки в разных языках?"
A: "Мы не проверяем синтаксис на фронтенде - код отправляется на сервер,
где компилятор сам находит ошибки и возвращает их пользователю."

Q: "Почему для Java нужно писать весь класс, а для Python достаточно одной строки?"
A: "Потому что так работают языки! Java требует обязательной структуры класса,
а Python можно писать в 'скриптовом' стиле."

Q: "Что происходит когда пользователь нажимает 'Выполнить код'?"
A: "Код + выбранный язык отправляются на наш Go-сервер, там запускаются в безопасной среде,
и результат возвращается обратно в интерфейс."
